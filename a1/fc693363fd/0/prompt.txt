### Strengths

**Architecture is clean and faithful to the plan.** The single-reducer pattern is correctly executed — all side effects stay outside the reducer, `useGameLoop` and `useInput` are proper thin wrappers, and state transitions are pure and testable.

### Issues

#### Critical (Must Fix)

None.

#### Important (Should Fix)

**Layout deviates from the spec.** The spec says `[Left Sidebar (score/level/lines)] [Board] [Right Sidebar (next piece + controls)]`. The implementation has a placeholder `&lt;div className="w-28" /&gt;` on the left and puts everything (score, level, lines, next piece, controls) in the right sidebar. The placeholder spacer balances the layout visually, but it wastes half the sidebar space and doesn't match the spec. This is low-risk for a demo but a clear plan deviation.

**`SOLID_CLASSES` is duplicated** between `/src/components/Board.jsx` and `/src/components/NextPiece.jsx`. Both define an identical object mapping color indices 1–7 to Tailwind classes. The canonical source already exists at `/src/constants.js` as `COLORS`, but `COLORS` maps `0 -&gt; null` not an empty-cell sentinel, so it can't be used directly. The fix is either to export `SOLID_CLASSES` from `constants.js` or from a shared `colorUtils.js`, not redefine it in two components.

#### Minor (Nice to Have)

**Dead import in `Board.jsx`.** `COLORS` is imported from `constants.js` but never referenced in the component body. The component uses its own `SOLID_CLASSES` and `GHOST_CLASSES` instead. Remove the `COLORS` import.

**Ghost piece renders at full border opacity, not ~30%.** The spec says "outlined, ~30% opacity cells." The implementation renders a solid-colored border with transparent fill (`border-2 border-{color} bg-transparent`). This is visually reasonable (a full-opacity outline clearly shows the ghost) but diverges from the "low-opacity" intent. Adding `opacity-40` to ghost cells would match the spec more closely without breaking the border approach.

**`SOFT_DROP` recomputes `ghostY` unnecessarily.** Moving down by 1 row doesn't change where the piece will land, so recomputing `ghostY` after a soft drop is redundant work. `TICK` correctly skips this recompute. Not a bug, but an inconsistency.

**`useCallback(dispatch, [])` with an empty dependency array** produces a React exhaustive-deps lint warning. `dispatch` from `useReducer` is already a stable reference — the `useCallback` wrapper is unnecessary but harmless. The lint warning can be avoided by removing `useCallback` entirely and passing `dispatch` directly to the hooks.
